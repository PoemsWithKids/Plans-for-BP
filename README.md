## 流程&规范
### 时间安排
平时主要依托github和微信进行异步讨论。周末进行线上会议。线上会议的讨论重点会提前在issue中提出。

### 规范
- issue:“卡片”层面的想法，主要用于集思广益和素材积累
- code：“篇章”篇章层面。已有基本成型的想法后可以发布在code中，团队成员在history进行讨论和修改。

code使用方法：
1.在仓库的code界面中找到要进行修改的文档，并点击
